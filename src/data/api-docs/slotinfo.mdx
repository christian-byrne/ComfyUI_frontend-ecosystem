Read-only snapshot of a single slot (input or output) on a node.

Identity is opaque per D20: use `slot.id` and `slot.equals(other)` for
comparisons; do not parse the string format.

## Properties

| Property | Modifier | Type | Description |
| ------ | ------ | ------ | ------ |
| <a id="direction"></a> `direction` | `readonly` | [`SlotDirection`](./slotdirection) | Whether this is an input or output slot. |
| <a id="id"></a> `id` | `readonly` | `string` | Opaque identifier for this slot. Treat as a string token; do not parse. |
| <a id="name"></a> `name` | `readonly` | `string` | Slot name as declared in `INPUT_TYPES` or `addInput`/`addOutput`. |
| <a id="nodeid"></a> `nodeId` | `readonly` | `string` | Opaque identifier of the node this slot belongs to. |
| <a id="type"></a> `type` | `readonly` | `string` | Slot type string (e.g. `'IMAGE'`, `'LATENT'`, `'*'`). |

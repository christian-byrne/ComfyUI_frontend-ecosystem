Payload for `widget.on('valueChange', handler)`.

Replaces the v1 `widget.callback` pattern.

```ts
widget.on('valueChange', (e) => {
  console.log('changed from', e.oldValue, 'to', e.newValue)
})
```

## Type Parameters

| Type Parameter | Default type | Description |
| ------ | ------ | ------ |
| `T` | [`WidgetValue`](./widgetvalue) | The widget's value type. |

## Properties

| Property | Modifier | Type | Description |
| ------ | ------ | ------ | ------ |
| <a id="newvalue"></a> `newValue` | `readonly` | `T` | Value after the change. |
| <a id="oldvalue"></a> `oldValue` | `readonly` | `T` | Value before the change. |

Controlled surface for widget access. Backed by ECS `WidgetValue` and
`WidgetIdentity` components in the World. Reads query components directly;
writes dispatch commands (undo-able, serializable, validatable).

All views (node, properties panel, promoted copy) share the same backing
`WidgetEntityId`, so mutations from any source trigger `valueChange`.

```ts
import { defineNode } from '@comfyorg/extension-api'

export default defineNode({
  name: 'my-extension',
  nodeCreated(node) {
    const steps = node.getWidget('steps')
    if (!steps) return

    steps.on('valueChange', (e) => console.log('steps =', e.newValue))
    steps.setOption('min', 1)
    steps.setOption('max', 150)
  }
})
```

| <a id="name"></a> `name` | `readonly` | `string` | The widget's name as registered in `INPUT_TYPES` or `addWidget`. Stable for the lifetime of the node; never changes after creation. |
| <a id="options"></a> `options` | `readonly` | `Readonly`\<[`WidgetOptions`](./widgetoptions)\> | Read-only snapshot of the full options bag for this widget. **Immutable per D-immutability-enforcement (Hybrid C).** The returned object is `Readonly<WidgetOptions>` — `widget.options.min = 0`, `widget.options = {...}`, and `widget.options.values = [...]` all raise TypeScript errors at compile time. To mutate, use [WidgetHandle.setOption](#setoption) per-key. Note: this is an accessor pair on the v2 surface. Reading is free; the setter intentionally does not exist on the public type. v1 patterns like `widget.options.serialize = false` should migrate to [WidgetHandle.setSerializeEnabled](#setserializeenabled); `widget.options.values = [...]` (combo refresh) migrates to a future `setValues` mutator (tracked under W6.P8.UNMIGRATABLE). **Example** `// ❌ TS-ERR — every option write raises a compile-time error widget.options.min = 0 widget.options = { min: 0, max: 100 } widget.options.serialize = false // ✅ Read freely const min = widget.options.min ?? 0 // ✅ Mutate via typed setters widget.setOption('min', 0) widget.setSerializeEnabled(false)` |
| <a id="serializevalue"></a> ~~`serializeValue`~~ | `readonly` | ((...`args`) => `unknown`) \| `undefined` | The widget's current `serializeValue` function (or `undefined` if none is registered). **Accessor-only per D-immutability-enforcement (Hybrid C).** The setter intentionally does not exist on the public type — assignment (`widget.serializeValue = fn`) raises a TypeScript error. The v2 migration target is the [\`on('beforeSerialize', fn)\`](#on) event (per D5), which is typed, async-capable, and composable across multiple extensions on the same widget. **Deprecated** v1 callers reading `widget.serializeValue` to invoke the function directly should subscribe to `'beforeSerialize'` instead. This read-only accessor exists for debugging / introspection only and may be removed once the v1 surface is fully retired. **Example** `// ❌ TS-ERR — direct assignment no longer compiles widget.serializeValue = () => 'static value' // ✅ Subscribe to the typed event (D5) widget.on('beforeSerialize', (e) => { if (e.context === 'prompt') e.setSerializedValue('static value') })` **Stability** experimental |
| <a id="widgettype"></a> `widgetType` | `readonly` | `string` | The widget's type string (e.g. `'INT'`, `'STRING'`, `'COMBO'`, `'MARKDOWN'`). Read-only invariant set at creation. |

### setHeight()

```ts
setHeight(px): void;
```

Defined in: [src/extension-api/widget.ts:366](https://github.com/Comfy-Org/ComfyUI_frontend/blob/59a2e1ce81b1ae31a7991a24d5b4ecb0a88d27ad/src/extension-api/widget.ts#L366)

Updates the reserved height for this widget and triggers a node relayout.

Meaningful for widgets registered via [defineWidget](./definewidget) with a
[WidgetMountFn](./widgetmountfn) `mount()` body — the reserved height bounds the
runtime-owned host `<div>` that the mount body renders into. For widgets
that render through the native widget renderer (no `mount`), this is a
no-op.

Replaces the v1 pattern of re-assigning `node.computeSize` to return a new
height whenever the embedded element resizes.

#### Parameters

| Parameter | Type | Description |
| ------ | ------ | ------ |
| `px` | `number` | New reserved height in pixels. |

Options for the global `defineExtension` entry point. Covers extension-wide
lifecycle and shell UI contributions.

## Stability

stable

```ts
import { defineExtension } from '@comfyorg/extension-api'

export default defineExtension({
  name: 'my-org.my-extension',
  async setup() {
    // App is ready; register commands, sidebar tabs, etc.
  }
})
```

## Properties

| Property | Type | Description |
| ------ | ------ | ------ |
| <a id="apiversion"></a> `apiVersion?` | `string` | Declared API version of this extension. Used by the telemetry system to track v1 → v2 adoption (D6 Phase D gate: "\<5% v1 usage before dropping the v1 bridge"). Set to `'2'` for extensions written against this API. Optional in Phase A (no runtime enforcement). The runtime reads this field via `getExtensionVersionReport()` to produce adoption metrics. **Stability** stable **Example** `defineExtension({ name: 'my-ext', apiVersion: '2', setup() { … } })` |
| <a id="name"></a> `name` | `string` | Globally unique extension name. Matches the format of `NodeExtensionOptions.name`. **Stability** stable |

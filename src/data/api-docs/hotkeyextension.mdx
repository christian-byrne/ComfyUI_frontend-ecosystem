Public arg type for [defineHotkey](./definehotkey). A hotkey binds a key combination
(already-registered command) to fire on press. Mirrors v1
`extension.keybindings[]` which used the internal `Keybinding` shape.

`keys` accepts a human-readable combo string (e.g. `'mod+k'`, `'ctrl+shift+f'`)
matching the Vue/PrimeVue keybinding convention. `mod` resolves to `cmd` on
macOS and `ctrl` elsewhere. The runtime parses this into the underlying
`KeyCombo` shape used by the keybinding store.

## Properties

| Property | Type | Description |
| ------ | ------ | ------ |
| <a id="commandid"></a> `commandId` | `string` | The id of an already-registered command. Use [defineCommand](./definecommand) to register the command before (or alongside) the hotkey. |
| <a id="keys"></a> `keys` | `string` | Key combination to listen for. Examples: `'mod+k'`, `'ctrl+shift+f'`, `'alt+enter'`. `mod` resolves to `cmd` on macOS and `ctrl` elsewhere. |
| <a id="targetelementid"></a> `targetElementId?` | `string` | Optional id of the DOM element that must be focused for the hotkey to fire. When omitted, the hotkey is global. |

# Subagent Prompt — PKG6: Wire Generated Docs into docs.comfy.org

You are implementing PKG6 from `plans/P2-extension-api-package.md`.

## Prerequisites

- PKG5 must be complete (`packages/extension-api/docs-build/mintlify/`
  contains generated MDX).
- Access to `Comfy-Org/docs` repo.

## Required reading

1. `plans/P2-extension-api-package.md` (section "PKG6")
2. `Comfy-Org/docs` repo structure (workspace symlink at `docs/`)
3. Their existing `mint.json` for nav structure conventions
4. Their existing "Custom Nodes" or "Extensions" section for content
   placement

## Deliverables

### Phase 1 — One-shot integration

1. Open a PR against `Comfy-Org/docs` adding:
   - `extensions/api/` directory containing the MDX files generated by
     PKG5
   - `mint.json` update to insert the nav snippet from PKG5 under an
     appropriate parent group
   - A new top-level "Extension API Reference" page or section
   - Cross-links from existing extension dev docs (e.g.
     `extensions/development.md`) pointing at the new API reference

2. PR description must include:
   - Link to `@comfyorg/extension-api` package
   - Link to source-of-truth file (`src/extension-api/index.ts`)
   - Statement that this page is auto-generated and manual edits will
     be overwritten

### Phase 2 — Continuous integration (separate PR)

3. Add a workflow to `ComfyUI_frontend/.github/workflows/`:
   `extension-api-docs-publish.yml`. Triggered by:
   - `extension-api-publish.yml` completing successfully
   - `workflow_dispatch` for manual updates

   Steps:
   - Checkout ComfyUI_frontend
   - Build docs via `pnpm --filter @comfyorg/extension-api docs:build`
   - Checkout `Comfy-Org/docs` (with cross-repo deploy key or PAT)
   - Sync `docs-build/mintlify/` to `Comfy-Org/docs/extensions/api/`
   - Open a PR with auto-generated title / body referencing the
     package version

## Acceptance criteria

1. PR against `Comfy-Org/docs` renders correctly in their preview build.
2. `docs.comfy.org/extensions/api/node-handle` (or equivalent) loads
   and shows API docs.
3. Search indexes the new content.
4. Phase 2 workflow successfully opens a PR on a manual `workflow_dispatch`.
5. The two repos can be kept in sync via the workflow with no manual
   copy-paste.

## What to defer

- Translating API docs to other languages — Mintlify supports i18n;
  separate decision.
- Customizing API doc pages with hand-written prose (Mintlify supports
  importing MDX into other MDX) — separate task per type.
- Polishing visual presentation (icons, callouts, custom components) —
  iterate after live.

## Verification commands

```bash
# Locally render the docs.comfy.org site with the new pages
cd docs/  # the Comfy-Org/docs symlink
pnpm dev  # or whatever Mintlify uses
# Visit http://localhost:PORT/extensions/api/
```
